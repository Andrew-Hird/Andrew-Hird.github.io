Technical Blog Part 2

Grid based design is breaking the layout of the page into different boxes. By doing this it is really easy to layout the page and work out what is going to go where and how to do it. A page might not always look like it is sitting on a grid, but if you break it down and draw boxes round the sections, you'll soon see that it is.


Responsive webpages are great because they'll look good no matter what size screen you are on. By elements moving closer together as the screen size gets smaller. This makes making pages for different types of devices a lot easier and still look good.


Semantic structure breaks the page up into different parts. As you can see below.
Each section has a different purpose of what information would go in it. For example the header area would usually house the name of the page or logo and the nav section would have any links you might use to get around the website.


I did find the process of setting up boxes and making a responsive webpage tricky to start with and had to follow quite a few instructions. I am finding that I am referring to guides quite a bit a this stage and am looking forward to when I can do most things by memory!
